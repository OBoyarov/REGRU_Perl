Ты вводишь в адресной строке браузера reg.ru, и через несколько секунд видишь главную страницу сайта. Опиши текстом, что происходит между этими событиями?

1. Браузер ищет в локальном кэше(History, DNS) IP-адрес домена.
2. Если IP-адрес есть, то пытается открыть сайт по этому адресу, в противном случае идет искать IP-адрес у DNS-кэша провайдера или с DNS-серверов.
3. При получении IP-адреса, браузер устанавливает соединение с серверов по протоколу TCP и в случае успеха браузер начинает отправлять запросы по протоколу HTTP(S).
4. Запросы от браузера (GET\POST) поступают на сервер, где их обрабатывает веб-серверная часть ПО (бэкенд) и возвращает обратно браузеру.
5. На основе полученных данных браузер либо отображает готовую страницу, либо производит дополнительные вычисления на основе скриптов js, css (фронтенд), или делает дополнительную отправку запросов(POST), например через API запрашивает баланс аккаунта - асинхронное обновление контента.
6. В зависимости от правил и заголовков браузер сохраняет в кэш статику сайта для последующей быстрой загрузки страницы.
